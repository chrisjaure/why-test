# What to test?

## Unit tests

- Utility methods
  - Give input, verify output
- Shared components
  - Different component states
  - Tricky logic

## Integration tests

- Time-intensive manual tests
  - Set up the scenario in your test
  - Computer is good at repetetive tasks
- Data-driven UIs
  - Verify different outcomes based on hard-coded API payloads
  - Data is often hard to find, put it in your test

## End-to-end tests

- Critical user flows
  - Can a user sign up?
  - Can a user perform an action?
